其中的泛型、结构体、Vec、usize我都能理解是什么？唯独 Arc<Box<[T]>> 这种实现一时没有看明白。感谢有 ChatGpt 的存在，复制过去直接进行询问，得到了清晰的解答，为此我整理了一篇知乎文章：解读：Arc> - 知乎。其中我认为对我关键的部分是：

  • [T] 是动态数组，表示一个长度未知的切片（slice）。
  • 因为要存储 任意长度的数组，必须用切片 &[T]，但切片本身没有所有权，因此使用 Box<[T]> 在堆上分配数据并获取所有权。
  • 使用 Arc<T> 是因为 Box<T> 只能有一个所有者，而如果 Tensor<T> 的多个实例需要共享 data ，Box<T> 则不够用。而 Arc<T> 则允许多个 Tensor<T> 共享同一个数据，并在最后一个 Tensor 被释放时才释放 data 。

Tensor 的方法

既然 Tensor 结构体字段已经解读明白了，还需要再理解 Tensor 都提供了哪些方法？

  • new ：是使用传入给定的 data 和 shape 创建一个 Tensor 实例。
  • default：是通过传入的 shape 参数来获得该形状的默认结构体实例。
  • data()：获取结构体 data 的动态数组数据的不可变引用。
  • data_mut()：则是获取结构体 data 的动态数组数据的可变引用。
  • shape()：获取结构体实例的 shape 字段。
  • size()：获取结构体实例的 length 字段。
  • reshape()：通过传入的 new_shape 来修改原有的结构体的 shape 。
  • slice()：则是通过传入的 start 参数和 shape 参数获取原有 Tensor 结构体的切片。
  • 针对 Tensor<f32> 特定类型的方法 print 打印结构体实例，这个方法对后面的调试很有帮助。
